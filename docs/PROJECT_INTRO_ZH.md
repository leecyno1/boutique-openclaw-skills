# 项目介绍：boutique-openclaw-skills

`boutique-openclaw-skills` 是一个面向生产环境的 OpenClaw skills 精选项目。它不是“技能市场镜像”，而是“运营可控的精选商店”：同类功能只保留一个最优技能，减少决策分叉，降低模型工具选择成本。

这个项目解决三类常见问题：

1. **重复能力**：多个技能做同一件事，导致 agent 误选、输出不稳定。
2. **Token 浪费**：能力重叠会放大工具规划链路，推理冗余增加。
3. **版本失控**：上游更新频繁，缺少审计时容易引入行为漂移和安全风险。

项目采用默认三档 + 兼容精选档策略，用 `tiers/low|medium|high.json` 维护默认 OpenClaw skills 的安装真相；`profiles/` 继续保留行业精选安装档。

为了保证长期可用，项目内置定期更新和风险审计机制：更新后自动检查能力去重、依赖缺失和高风险代码模式，生成本地审计报告，确保技能组合持续可靠、可追踪、可维护。

一句话总结：**它是 OpenClaw 的“技能编排中台”，目标是稳定可用，而不是功能堆叠。**
