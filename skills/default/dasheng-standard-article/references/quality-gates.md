# Quality Gates（标准文章硬门禁）

## 1) 内容门禁（必须全过）

- 字数：`>= 5000`（按非空白字符计）
- 数据表：`>= 2` 张
- 来源：`>= 3` 条可访问来源
- 正文结构完整：标题 + 问题定义 + 事实层 + 机制层 + 情景推演 + 执行建议 + 结论 + 来源
- 段落自然：每个 H2 正文不超过 5 段（避免墙式堆叠）

## 2) 禁出清单（默认不允许）

以下内容默认不得出现在交付正文中：

- 口播脚本 / 视频脚本
- 标题候选
- “如何写”“写作方法”“工作流说明”
- brief/run_id/workflow/模型参数等内部流程痕迹
- “该章节为固定结构补位”之类模板占位语

## 3) 飞书写入门禁

覆盖写入后必须立即回读，确认：

- `revision_id` 已更新
- 文档 `block_count` 明显大于空文档
- 文本内容为完整文章，而非提纲/说明书
- 未出现乱码或异常字符

## 4) 失败处理

- 若质检失败：修正文稿，重跑质检。
- 若飞书回读失败：再次覆盖并回读，直到通过。
- 未通过前禁止对用户说“已完成”。

## 5) 用户优先级

当用户明确说“初稿就是完整标准化文章”时：

- 自动禁用所有衍生稿（口播/视频/标题候选）
- 只保留单篇完整正文交付
