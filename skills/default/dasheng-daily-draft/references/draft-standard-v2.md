# Draft Standard v2

## Scope

适用于大圣工作流“初稿环节”所有默认产出，目标是直接服务公众号主文与后续文案/口播转写。

## Hard Requirements

1. 字数要求：每篇 `>=5000`（非空白字符）。
2. 结构要求：必须包含以下段落：
   - `开篇立论`
   - `核心论证`
   - `数据证据与论据`
   - `关键数据表`
   - `策略与执行建议`
   - `数据与素材来源`
3. 纯正文要求：
   - 禁止内部生产术语（如 `brief`、`run_id`、`提示词`、`模型参数`、`采集流程`、`SOP群`）。
   - 禁止引用 intake/工作流内部统计词（如 `热度`、`扩散`、`S/A占比`、`流程统计`、`内部统计`）。
   - 语体必须为标准文章表达，可直接用于公众号，且不得注入风格 DNA。
4. 段落要求：
   - 每个 `H2` 章节正文段落不超过 `5` 段。
5. 证据要求：
   - 必须包含可核验数据点（数字证据）。
   - 每篇至少 `3` 条来源口径。
   - 证据仅可来自“底稿链接素材”或“外部公开数据源”（如 `akshare`、`tushare`、`新浪财经`、`人民网`、`CNN`）。
   - 禁止使用占位来源（如 `example.com/source-pending`）。
6. 表格要求：
   - 每篇至少 `2` 张 markdown 表格。
   - 表格需支持核心结论，不得仅装饰。
7. 转写要求：
   - 自动输出转写策略，固定 `rewrite_model_mode=bm`。
8. 交付要求：
   - 本地留存初稿文件。
   - 飞书文档可读、非空。
   - SOP 群同步交付报告。

## Quality Checklist

- 观点主线是否单一清晰。
- 论据是否形成“观点-证据-动作”闭环。
- 表格是否直接支撑结论。
- 每节是否满足段落上限。
- 是否命中禁用术语（命中即失败）。
- 结尾是否给出可执行下一步。
- 来源是否可追踪。

## Delivery Checklist

- 稿件标题清单
- 每篇字数
- 每篇飞书链接
- 群同步 message_id
- 转写模式（必须为 `bm`）
- 交付报告链接（可选）
