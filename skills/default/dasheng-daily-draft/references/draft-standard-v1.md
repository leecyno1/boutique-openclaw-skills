# Draft Standard v1

## Scope

适用于大圣工作流“初稿环节”所有默认产出，目标是直接服务公众号主文创作。

## Hard Requirements

1. 字数要求：每篇 `>=3000`（非空白字符）。
2. 结构要求：必须包含以下 5 段（可扩展但不可缺失）：
   - `核心结论`
   - `关键数据与证据`
   - `关键数据表`
   - `策略与执行建议`
   - `数据与素材来源`
3. 证据要求：
   - 必须包含可核验数据点。
   - 每篇至少 `3` 条来源口径。
4. 表格要求：
   - 每篇至少 `1` 张 markdown 表格。
   - 表格至少包含：指标、数据/现象、含义（或同等字段）。
5. 交付要求：
   - 本地留存初稿文件。
   - 飞书文档可读、非空。
   - SOP 群同步交付报告。

## Quality Checklist

- 观点主线是否单一清晰。
- 数据是否服务论证，而非装饰。
- 表格是否支持关键结论。
- 结尾是否给出可执行下一步。
- 来源是否可追踪。

## Delivery Checklist

- 稿件标题清单
- 每篇字数
- 每篇飞书链接
- 群同步 message_id
- 交付报告链接（可选）
